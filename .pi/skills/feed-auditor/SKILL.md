# Feed Auditor

For the Feng Wen (风闻) project at `/workspace`. Audits all feed generators
against their live source websites, using `feeds/_audit.json` as a cheap
first-pass triage.

## Quick Start

```bash
# Regenerate the audit report (if stale)
cd /workspace/src && python3 audit_report.py
```

Then follow the audit workflow below.

## Audit Workflow

### Phase 1: Triage from `_audit.json`

Read `feeds/_audit.json`. It contains a pre-computed health check of every
feed generated by the most recent CI run.  Sort findings by priority:

| Priority | Issue | Meaning |
|----------|-------|---------|
| **P0** | `xml_missing` | Output file was not generated at all — script likely crashed or CI skipped it |
| **P0** | `xml_parse_error` | XML file exists but is malformed — script produced garbage |
| **P1** | `zero_entries` | Script ran successfully but extracted nothing — selectors likely broken |
| **P1** | `all_same_date` | Every entry has today's date — all dates fell back to `datetime.now()`, real dates not parsed |
| **P2** | `stale_Nd` | Newest entry is N days old — may be legitimate (site is dormant) or a broken selector missing new posts |
| **Info** | `unchecked_official` | External RSS/Atom feed not yet verified — these are listed in the report for awareness |

Feeds with status `healthy` can generally be trusted, but spot-check a random
sample (see Phase 3).

### Phase 2: Investigate each warning/error feed

For each feed with issues, investigate the live source.  The approach depends
on the feed's strategy (recorded in `_audit.json`):

#### `scrape` feeds (HTTP scraping with BeautifulSoup)

```bash
# 1. Fetch the page to see what the script gets
cd /workspace/src && python3 -c "
from utils import fetch_page
html = fetch_page('PAGE_URL_HERE')
print(len(html), 'bytes')
" > /tmp/page.html

# 2. Check for signs of trouble:
#    - 403 / Cloudflare block
#    - Redirect to new URL
#    - Site redesign (compare selectors in the script to actual HTML)
#    - Content still exists but under different CSS classes
```

Read the corresponding `scrape_*.py` script.  Note the CSS selectors it uses.
Open `/tmp/page.html` and check:
- Do the selectors still match anything?  `grep` or use Python/BeautifulSoup
- Is the content structure recognizable?  Look for article cards, list items
- Are there NEW articles on the page that the feed is missing?

#### `request` feeds (async API/JSON/RSC fetching)

```bash
# Fetch the API endpoint and inspect the response
cd /workspace/src && python3 -c "
from curl_cffi.requests import Session
s = Session()
resp = s.get('API_URL_HERE', impersonate='chrome120')
print(resp.status_code)
print(resp.text[:2000])
"
```

Check: did the API change format?  New fields?  Pagination changes?  401/auth errors?

#### `crawl` feeds (browser-based with Playwright)

These are the hardest to debug — they require a real browser:

```bash
cd /workspace/src && python3 -c "
from playwright.sync_api import sync_playwright
with sync_playwright() as p:
    browser = p.chromium.launch()
    page = browser.new_page()
    page.goto('PAGE_URL_HERE')
    page.wait_for_timeout(3000)
    html = page.content()
    print(len(html), 'bytes')
    # Check if expected elements exist
    import re
    matches = re.findall(r'EXPECTED_PATTERN', html)
    print(f'Found {len(matches)} matches')
    browser.close()
"
```

#### `enhance` feeds (official feed repair)

```bash
# Fetch the upstream feed directly
cd /workspace/src && python3 -c "
from utils import fetch_page
xml = fetch_page('UPSTREAM_FEED_URL')
print(xml[:2000])
"
```

Check: has the upstream feed moved?  Changed format?  Is it still publishing?

### Phase 3: Spot-check "healthy" feeds

Pick 3–5 feeds marked `healthy` at random.  For each:
1. Fetch the source page
2. Spot-check that at least one entry from the XML appears in the page HTML
3. Verify dates in the XML look reasonable (not all today, not all ancient)
4. If anything looks off, escalate to full investigation

### Phase 4: Report

Produce a categorized summary:

```
## Feed Audit Report

### Critical (0 entries / broken)
- feed_name: selector `.post-card` no longer matches (site redesign on 2026-07-01)

### Needs attention (stale / suspicious)
- feed_name: 285d stale — site hasn't published since 2025-09, feed is correct

### Healthy (requires no action)
- 49 feeds verified via _audit.json + spot checks

### Official feeds not yet checked
- N feeds — suggest checking top 5 by importance
```

## Key Files

- `config/feeds.json` — canonical feed inventory
- `feeds/_audit.json` — machine-generated health report (updated by CI)
- `feeds/*.xml` — output Atom feeds
- `src/scrape_*.py` — HTTP scraping scripts
- `src/request_*.py` — async API/JSON/RSC scripts
- `src/crawl_*.py` — browser-based scripts
- `src/enhance_*.py` — official feed repair scripts

## Prerequisites

- Python 3.12+ with `curl_cffi`, `beautifulsoup4` installed
- For crawl feeds: `playwright` with bundled Chromium (install via `playwright install chromium`)
- Run from `/workspace` with `PYTHONPATH` set to `src/`
